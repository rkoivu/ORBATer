## QOL Improvements Summary

This commit implements 5 high-value quality-of-life improvements focused on user feedback, discoverability, and error clarity:

### Changes

1. **Icon Mode Toggle Feedback** (symbols-data.js)
   - Users now see toast confirmation when enabling/disabling NATO symbols
   - Prevents confusion when icons unexpectedly disappear
   - One-line change, zero breaking risk

2. **Tab Dirty State Indicators** (views-snapshots-command-palette.js)
   - Shows `●` prefix on tabs with unsaved changes
   - Users immediately see which tabs need saving
   - Prevents accidental data loss in multi-tab workflows
   - Per-tab dirty state tracking with automatic clear on save

3. **Keyboard Shortcuts Discovery** (qol-keyboard-feedback.js - new module)
   - New "⌨ Shortcuts" button in toolbar opens comprehensive shortcuts modal
   - Ctrl+Shift+? keyboard shortcut for quick access
   - 10 most-used shortcuts displayed with clear descriptions
   - Tooltip reminder that toolbar buttons show shortcuts on hover

4. **Enhanced Import Error Messages** (qol-keyboard-feedback.js)
   - Specific error feedback instead of generic "failed"
   - Distinguishes: empty file, invalid JSON, missing units, unsupported format
   - Users understand what went wrong and can take corrective action
   - Console logs full error details for debugging

5. **New QOL Module** (qol-keyboard-feedback.js)
   - Modular, non-intrusive IIFE-based additions
   - No modifications to existing core logic
   - Defensive programming with try-catch throughout
   - Graceful degradation if dependencies unavailable
